# Local Agent Notes

- 모든 사용자 대상 응답은 기본적으로 한글로 작성한다.
- 사용자가 영어를 명시적으로 요청한 경우에만 영어로 전환한다.
- 저장소 안에 문서나 설명 파일을 새로 만들 때도 특별한 이유가 없으면 한글을 우선 사용한다.
- GitHub 이슈 제목과 본문은 기본적으로 한글로 작성한다.
- GitHub PR 제목과 본문은 기본적으로 한글로 작성한다.
- GitHub 리뷰 코멘트, 일반 코멘트, 답글은 기본적으로 한글로 작성한다.
- `gh` CLI를 사용할 때 자동 생성 문구가 영어면 한글로 직접 덮어쓴다.
- 일반 Git 커밋 메시지도 기본적으로 한글로 작성한다.
- 머지 커밋 제목과 설명도 특별한 이유가 없으면 한글로 맞춘다.
- 코드 식별자, 라이브러리 이름, 명령어, 고유명사는 필요할 때 영어를 그대로 쓴다.
- 작업은 가능한 한 작업 단위별로 로컬 커밋을 남긴다.
- 푸시하기 전에는 현재 작업과 직접 관련 있는 유효한 커밋만 남도록 커밋 히스토리를 정리하는 것을 우선한다.
- 이미 원격에 푸시된 커밋이나 다른 사람이 만든 커밋은 사용자의 명시적 요청 없이 재작성하지 않는다.
## 파일 탐색 규칙

- 저장소 전체 파일 목록이 필요할 때는 `rg --files .`보다 `git ls-files`를 우선 사용한다.
- README처럼 이름이 명확한 파일을 찾을 때는 `Get-ChildItem -Path . -Filter "README*" -File`처럼 범위를 좁힌다.
- 검색 전에 `.rgignore`를 존중하고, 대용량 산출물 디렉토리로 탐색이 새지 않게 한다.
