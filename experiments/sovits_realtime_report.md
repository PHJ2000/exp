# so-vits 실시간성 실험

## 조건
- 날짜: 2026-03-25
- GPU: NVIDIA GeForce RTX 4060 8GB
- CUDA: torch 2.9.1+cu128
- 모델: `therealvul/so-vits-svc-4.0` 의 `Pinkie (speaking sep) / G_166400.pth`
- 스피커: `Pinkie {neutral}`
- 입력 음성: 저장소에 포함된 `LJ001-*.wav` 샘플을 이어붙여 1초, 3초, 6초, 12초 클립 생성
- 워크로드: `so-vits-svc-fork` 의 `Svc.infer_silence` 와 `RealtimeVC2.process`

## 방법
1. 모델과 ContentVec를 로드한다.
2. 1초 무음으로 한 번 워밍업한다.
3. 오프라인 변환의 실시간 계수(RTF = 처리시간 / 오디오길이)를 잰다.
4. 12초 입력을 0.5초, 1.0초 블록으로 잘라 `RealtimeVC2` 스트리밍 경로를 시뮬레이션한다.

## 결과

### 프로세스 시작 후 첫 출력까지
- 모델 로드: 3.482초
- 첫 추론: 0.858초
- 프로세스 시작부터 첫 오디오까지: 4.340초

### 오프라인 변환, 워밍업 이후
- 1초 입력: 0.513초, RTF 0.513
- 3초 입력: 0.674초, RTF 0.225
- 6초 입력: 0.697초, RTF 0.116
- 12초 입력: 1.085초, RTF 0.090

### 스트리밍 경로, 워밍업 이후
- 0.5초 블록: 평균 RTF 0.106, 중앙값 0.021, p95 0.660, 최대 1.304, 실시간 초과 블록 1/24
- 1.0초 블록: 평균 RTF 0.090, 중앙값 0.012, p95 0.465, 최대 0.636, 실시간 초과 블록 0/12

## 해석
- 이 환경에서는 워밍업 이후 기준으로 `거의 실시간` 주장은 대체로 성립한다.
- 다만 0.5초처럼 짧은 블록에서는 간헐적으로 `RTF > 1` 스파이크가 나온다.
- 그래서 안정적으로 말하려면 `항상 완전 실시간` 보다는 `1초 안팎 블록에서는 실시간에 가깝고, 0.5초 블록에서는 가끔 밀릴 수 있음` 이 더 정확하다.
- `빡센 파인튜닝` 자체가 속도를 올려준다고 보기는 어렵고, 이번 결과는 주로 모델 구조와 GPU 추론 성능을 본 것이다.
